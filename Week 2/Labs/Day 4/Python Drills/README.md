# Python Drills (Intermediate): Foundations and the Standard Library

**Difficulty:** Intermediate  ·  **Format:** Individual  ·  **Estimated time:** 60 to 90 minutes (self-paced)

This is a slow, confidence-building rep set that reinforces the Python from Day 2 and
adds the standard library modules data engineers reach for every day. It is a single
large notebook you work through top to bottom. It complements the main Day 4 medallion
lab: here you do grouping, date parsing, and file discovery by hand, so that when
`pandas` does it for you later, you know what is happening under the hood.

> **AI-Free Zone (Weeks 1 to 4).** Complete these drills yourself. Do not use Copilot,
> ChatGPT, or any AI assistant to write the code. Use the linked official docs and the
> collapsible hints. Calling an AI service through an API is a different thing (and is
> fine); typing your practice code for you is not.

## What you will practice

**Foundations:** building functions (parameters, defaults, docstrings, `return`),
`for` loops (`range`, `enumerate`, `zip`, running totals), and nested data structures
(list of dicts, dict of dicts, safe access with `.get`).

**Standard library modules:** `math`, `statistics`, `random`, `datetime` +
`timedelta`, `calendar`, `pathlib`, `glob`, and `itertools`. Each module drill starts
with a plain explanation, a link to the official docs, one or two runnable examples,
then a task you complete on your own.

**Capstone challenge:** a tiny by-hand "monthly report" pipeline that combines
`glob` / `pathlib`, `csv`, `datetime`, `statistics`, and `itertools`.

## Setup

Follow the same environment you set up in `Activity_0_UV_GCS_Setup.md` for Day 4.
These drills use only the Python standard library, so there is nothing extra to
install.

1. From the repo root, copy this whole `Python Drills` folder into your day project so
   you never edit the provided copy in place (this keeps `git pull` from conflicting
   with your work):

   ```bash
   mkdir -p student-work/week2/day4
   cp -r "Week 2/Labs/Day 4/Python Drills" student-work/week2/day4/
   ```

2. If you have not created the Week 2 UV project yet:

   ```bash
   cd student-work/week2
   uv init            # only if you have not already; creates pyproject.toml and .venv here
   ```

   `.venv` lives at `student-work/week2/.venv`. `uv run` targets it automatically. If
   you see a `VIRTUAL_ENV does not match the project environment` warning, a stray env
   from another folder is active: run `deactivate`, then optionally
   `source .venv/bin/activate`.

3. Open `student-work/week2/day4/Python Drills/Python_Drills.ipynb` in VS Code and
   select `student-work/week2/.venv/bin/python` as both the VS Code interpreter and the
   notebook kernel (Select Kernel, top right of the notebook).

4. Add `.venv/`, `__pycache__/`, and `sample_data/` to the `.gitignore` in
   `student-work/week2/`. The `sample_data/` folder is generated by the notebook's
   first cell, so it does not belong in version control.

5. Run the first (Setup) cell before anything else. It creates `sample_data/` next to
   the notebook and defines the shared `CLAIMS` data.

## How to work through it

- Read the **Concept** for each drill and skim the linked docs.
- Run the **Example** cells and read the output.
- Complete the **starter** cell by replacing every `# TODO`.
- Run it and compare against the expected output noted in the task or comments.
- Stuck for more than a few minutes? Open the hint. Stuck longer? Peek at
  `solutions/` for that one drill, understand it, then close it and retype.

## Success criteria

- Every `# TODO` is implemented and every cell runs without error.
- Your outputs match the expected values called out in the tasks (for example
  `num_batches` is 5, `total_rows` is 8, and the capstone report totals match).
- You can explain, in one sentence each, what `strptime` vs `strftime` do, why you
  seed `random`, and why `itertools.groupby` needs sorted input.
